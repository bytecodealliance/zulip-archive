khagankhan commented on issue #12892:

Well it turns out after libfuzzer's own mutation one type may appear in two rec groups. While we had an assertion for that in is_well_formed, we never actually enforced it during fixup. Now it is fixed with a dedup pass in _step 2_:

let mut seen = BTreeSet::new();
for members in self.rec_groups.values_mut() {
    members.retain(|tid| self.type_defs.contains_key(tid) && seen.insert(*tid));
}

Addressed in the PR: #12900
