  • Add CExternType::Tag(CTagType) variant; wire into CExternType::new() and wasm_externtype_kind()
  • Add wasmtime/types/tag.hh C++ wrapper (TagType / TagType::Ref)
  • Extend ExternType::Ref variant and ref_from_c() switch in extern.hh
  • Add TagType tests: construction, module export enumeration (regression for #10252), and wasm_externtype_kind / cast function correctness

alexcrichton created PR review comment:

For the signature of this function I'd recommend following the same rough signature of Wasmtime itself. Namely in Wasmtime a tag is constructed from a full FuncType, whereas here all the pieces are provided.

In the original exception-handling proposal tags could only have parameters, but in the stack switching proposal this was relaxed to additionally allow result types too. Given that, and looking forward, I think we'll want to support constructing tags both with and without params/results, so at the very least this constructor needs to grow parameters to communicate the tag's result types. At that point I think it's probably easiest to just take a function type in the constructor.
