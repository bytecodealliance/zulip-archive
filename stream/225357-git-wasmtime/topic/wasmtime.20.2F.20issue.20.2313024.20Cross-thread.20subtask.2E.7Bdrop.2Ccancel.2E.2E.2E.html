TartanLlama commented on issue #13024:

Seems to me there are two issues here:

  • The thread index check should be a task index check
  • Even with the task index check, it's feasible that a different task in the same instance could just pull a handle index out of thin air and try to cancel it, in which case this should return an Err rather than asserting

Does that sound right to you @alexcrichton @dicej ?
