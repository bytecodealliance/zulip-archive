  • This fix is necessary because zero-length data frames are valid per the http_body::Body trait contract (which does not forbid them) and RFC 9113 §6.1 (which explicitly permits zero-length DATA frames), so HostBodyStreamProducer must tolerate them as a consumer of poll_frame.
